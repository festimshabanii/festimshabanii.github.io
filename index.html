Task Management

Technologies: IntelliJ
Programming Languages: Spring boot with JPA and Thymeleaf
Scripting languages: JavaScript & jQuery
Frontend: Bootstrap
Database: Postgres

Task management, it's a system for managing tasks of X company.
System is able to increase efficiency in work, based on UI and UX managing of tasks and drag & drop feature.
Base features:
  • 1. Creating a task: you have to fill all required fields
    • Type: specifies the importance of the task
    • Name: Task name
    • Description: Description of task
    • Assigne: You have to choose one of your employees for that specific task and after creation, the system will trigger a notification (email) for the employee that is in "Assignee" field
    • Reporter: is used to know who is creator of the task
    • Due Date: is for deadline of task
    • Time spent: After creation of the task, user that is assignee on that specific task can modify the "Time Spent" field for information of hours that have been spent on it

  • 2. Main view
    • After creation of task, task will appear on section of "Open phase" with ID and Priority included on header of the task and in content will have the Name of the task
    • System supports drag & drop feature for dragging task in to another phase and after task is dropped, system will trigger email notification for user that is "Assignee" for that task

  • 3. Emails content

e-document

Technologies: IntelliJ
Programming Languages: Spring boot with JPA and Thymeleaf
Scripting languages: JavaScript & jQuery
Frontend: Bootstrap
Database: Postgres

E-document, serves to create application/document inside of system automatically

  • 1. Main view
    • After login, we'll be able to see every application/documentthat we have created

  • 2. Creation of application / document
    • We have to specify one unique name and number for our new application/document and choose one of icons for application properties.

  • 3. E-document
    • System supports drag & drop feature for dragging fields in layout of our new application/document
    • Fields that can be inserted in layout of new application are: Text field, Dropdown list, Check box, Label, Date & time, Text area
    • After fields are dropped in new layout they can be editable by Field Details section in the right of the page
    • For Text Fields, Text Area, Date & Time - System supports attributes as placeholder, width, height, font size and color of text to be editable
    • For Label - System supports attributes as text, width, height, font size and color of text to be editable

  • 4. Files that will be created automatically
    • After button Save Document is pressed, system will start to create seven (7) files in backend for creation of new application with all fields and properties that we have specify
    • All seven (7) files are connected with each other:
    • Model class - serves for creation of new table in database based on the application name that we have specified in the begin. Fields that we have dragged and dropped in our new layout of application, will create new variables in model class and columns in table (database)
    • Service, Repository, Controller - Serves for functionalization of backend for storing/searching valuesfrom HTML files that system has created
    • HTML File (f86_fshabani.html) - Is the same as we have designed in E-Document section and it serves for storing values in table (database) that system has created
    • HTML Search File -Is new layout for report of our stored records with all fields that we have dragged and dropped in e-document included
    • Default fields: System will create two default fields automatically:
      • Files - This field support every type of files to be included and to store it in database
      • Responsible Person - This fields takes the ID of person that is creating new Record in our new Application and store it in database

  • 5. Database Table view

  • 6. Application that we have created

  • 7. Layout of Application/document that we have designed

  • 8. New layout that is generated by system for Reports
    • URL column - has included one link with name of the file that we have inserted in section 7.Layout of Application/document
      If we click on the link the same file that we have inserted before will be downloaded

  • 9. Filter Reports
    • Filter fields - We can filter which field we want to be shown on our Report table
    • Search field - We can search for specific record based on every field that is included in table

Schedule Management

Technologies: IntelliJ
Programming Languages: Spring boot with JPA and Thymeleaf
Scripting languages: JavaScript & jQuery
Frontend: Bootstrap
Database: Postgres

Schedul management, is used for storing information of employees on calendar (vecations and business hours with start and end time)
  • 1. Storing and Update information
    • First we have to select user in the dropdown list on the right of the page to get all the data for that specific user from database in our calendar
    • Store/Update schedule - After the data are passed on calendar we are able to click one of the cells in the table and the system can detect on what date we want to store or update information of the specific employee that we have selected before
    • We are able to select what type of information we want to store on database:
      • Work - After "Work" value is selected, system will display two fields: From & To.
        From field will store the specific hour when employee starts to work
        To field will store the specific hour when employee has finished
      • Vecation
    • Comment - in case if we want to put a comment for that specific day
    • System will difference vecations and working hours in calendar by colors
    • We can change the month by pressing the "right sign" or "left sign" next to Name of the month

  • 2.Report Section
    • Monthly Report
      • System is able to generate monthly reports for every user with all data for business hours and vecations
      • If we hover the cell, the comment of that user in that specific day of the month will be shown
      • We can change the month by pressing the "right sign" or "left sign" next to Name of the month

    • Weekly Report
      • System is able to generate weekly reports for every user with all data for business hours and vecations
      • We have to specify one date in the date field From and system will generate all the data from that specific day of the month till 6 more days

    • Specific Report
      • System is able to generate specific reports for specific user with all data for business hours and vecations
      • We have to select for which user we want to see the data on dropdown list in the right of the page

School Management System

Technologies: Netbeans
Programming Languages: JavaFX - library of Java
Frontend: FXML - XML-based user interface markup language | SceneBuilder Application
Database: MS SQL

School Management System - This system will be used to digitize and automate school processes. It's easy to use and it's user friendly based on JavaFx technology that uses fxml for better UI and UX
System works with three (3) type of users: Administrator, Teacher, Student
  • 1. Login form
    • The system will know if the user is a student or teacher or administrator based on the regex that the system uses

  • 2. Administrator Dashboard
    • Has four (4) information windows for:
      • 1. Number of Students
      • 2. Number of Teachers
      • 3. Number of Staff
      • 4. Number of Classes

  • 3. Administrator Email
    • The system supports the email feature that is manually made to operate within the system
      • 1.Inbox - is used to show all messages that are sent to specific user that is logged in
      • 2.Sent Mail - is used to show all messages that are sent from specific user that is logged in
      • 3.Compose - is used for writing new emails to someone that has user in our system

  • 4. Administrator Dashboard
    • Has two (2) main buttons for reports and one (1) for generating them in excel file
      • Report for teachers - is used to indicate when teachers have logged in and when they have logged out of the system
      • Report for workers - is used to indicate when workers have logged in and when they have logged out of the system

  • 5. Teachers button in admin form
    • Has one main table with all data of teachers
    • Three (3) buttons for Creating, Update,Delete users (Teachers)
    • Two (2) buttons for Exporting data that are in the table, in Excel file and PDF file

  • 6. Admin Settings
    • System will show all private information of the user that is logged in
    • All users can change their profile picture

    • All users can change their password

  • 7. Staff button in admin form
    • Administrator can see all staff by pressing "right button" and "left button"

  • 8. Student button in admin form
    • In this view we have one (1) main table that will show us all students that are registered in the system
    • Three (3) buttons for Creating, Update,Delete users (Students)
    • Two (2) buttons for Exporting data that are in the table, in Excel file and PDF file

  • 9. Teacher Dashboard
    • In this view we have all personal information of teacher that is logged in

  • 10. Class button in teacher form
    • This view will show us all classes that teacher is responsible for.
    • By clicking one of the classes that are in main table, system will pop up one window with all students that are in that specific class

  • 11. Mark button in teacher form
    • In this view, system will show us all marks that are inserted from logged in teacher
    • Every teacher can insert, edit, update and delete their marks

  • 12. Student button in teacher form
    • The system will display all the students of the teacher who is logged into the system

  • 13. Email feature in teacher form (manually made to operate within the system)

  • 14. Student dashboard
    • System will display all information of user (student) that is logged in

  • 15. Student settings
    • All users (students) can change their password

  • 16. Student marks
    • System will display all marks of specific user (student) that is logged in

AES - symmetric block cipher

Technologies: Netbeans
Programming Languages: JavaFX - library of Java
Frontend: FXML - XML-based user interface markup language | SceneBuilder Application

This program uses one of the most popular encryption standard of the world.
We will use the AES Algorithm for transformation of plain text in ciphertext (encrypted text) and the opposite.
AES is a symmetric-key algorithm, meaning the same key is used for both encrypting and decrypting and our key in this example is: ShabFest@1211#.,
  • 1. Encryption Function - We have written our plain text: festim shabani

  • 2. Encryption Function #2
    • After we pressed the button Encrypt, system will display another field with encrypted text

  • 3. Decryption Function
    • After we pressed the Decrypt button the transformation of ciphertext will be shown on the field of plain text
